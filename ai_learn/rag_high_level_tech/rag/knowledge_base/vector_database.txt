向量数据库介绍
============

什么是向量数据库？
向量数据库是专门用于存储和检索高维向量的数据库，是RAG系统的核心组件。

向量的生成：
文本通过Embedding模型转换为向量。常见的Embedding模型包括：
- OpenAI text-embedding-ada-002（1536维）
- sentence-transformers系列（768维）
- BGE系列（1024维）

语义相似度：
向量将文本转换为数字表示，使得语义相似的文本在向量空间中距离更近。
例如：
- "猫"和"狗"的向量距离较近（都是动物）
- "猫"和"汽车"的向量距离较远（不同类别）

常见的向量数据库：
1. Faiss
   - 开源（Facebook）
   - 极快的检索速度
   - 支持十亿级向量
   - 需要自己管理

2. Pinecone
   - 商业云服务
   - 易于使用
   - 自动扩展
   - 按使用付费

3. Chroma
   - 开源
   - 易于集成
   - 适合中小规模
   - 支持持久化

4. Weaviate
   - 开源
   - 支持多种数据类型
   - 内置向量化
   - GraphQL接口

Faiss索引类型：
1. IndexFlatL2：精确搜索（慢但准确）
2. IndexIVFFlat：近似搜索（快，需要训练）
3. IndexPQ：压缩索引（省内存）

向量检索流程：
1. 将查询文本转换为向量
2. 在向量数据库中搜索最相似的向量
3. 返回相似度最高的Top-K个文档
4. 将这些文档用于RAG生成

性能优化：
- 批量处理向量
- 使用GPU加速
- 选择合适的索引类型
- 定期重建索引
